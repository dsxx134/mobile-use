# Xianyu Location Writeback and Log Consolidation Design

Date: 2026-03-17

## Design Summary
This change addresses two friction points in the current Xianyu publish flow: location selection stalls because the editor still exposes `选择位置` after a search, and Bitable logging is split across too many fields. We will loosen location panel detection to rely on the stable header/section labels (`宝贝所在地`, `搜索地址`, `常用地址`) instead of requiring `请选择宝贝所在地`. Location selection will capture the tapped search-result text, normalize it into a single-line value, and treat that captured value as proof of a successful selection even if the editor still shows `选择位置`. The runner will write that normalized value back to the `预设地址` field when marking the listing `已就绪`, so the selected address is visible and reused for later runs. Failure artifacts remain captured locally, but Bitable writeback is consolidated to the single `失败原因` field by ignoring the separate failure-path columns. This keeps the Feishu table concise while preserving detailed artifacts on disk and in the log entry text. Error handling stays strict: if search results never appear or the selected target has no usable text, the flow raises and the row is marked failed. Testing will cover the relaxed panel detection, the new search helper returning a value, the updated location verification behavior, runner writeback of the selected address, and the absence of failure-artifact columns in Bitable updates.
