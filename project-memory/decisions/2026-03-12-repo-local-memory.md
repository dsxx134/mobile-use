# Use Repository-Local Memory

Date: 2026-03-12

## Decision
- Store long-term project memory inside this repository instead of `C:\Users\...`.
- Treat pushes as milestone checkpoints for meaningful project progress.

## Rationale
- Repo-local memory travels with the codebase.
- GitHub push history becomes part of the durable project record.
- This avoids scattering project context across machine-specific directories.
